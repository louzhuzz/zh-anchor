# 中文先行思考协议

## 核心锚定指令

```
当用户开启一个新会话、发送第一条消息时，必须先用中文写一段简短总结再开始任何推理或操作。
总结内容包括：用户让你做什么、你计划用什么步骤完成、你还缺什么信息。
这段中文是你第一个输出，完成这段输出后才能执行具体任务。
后续消息不受此限制，但须保持中文思维。
```

## 原理

利用自回归语言模型的因果依赖特性——**首个主动输出语言锚定整场对话**。

语言锚定效应已被多个独立验证：
- Contextual Inertia（上下文惯性）：模型在多轮对话中倾向保持初始模式
- Prefix Anchoring（前缀锚定）：强制模型以特定语言前缀开始推理可显著改善跨语言表现

## 与机械提示词的区别

| 方式 | 示例 | 效果 |
|---|---|---|
| 机械规则 | "禁止使用英文、必须用简体中文、禁止滥用标点…" | 多轮后偏离 |
| 文化语境 | "请用中文语言思维完成所有任务" | 持续稳定 |

文化语境框架以少量 token 激活预训练中广泛的中文思维模式，信息密度远高于机械规则列表。

## 使用方式

### 方式一：通过 OpenCode 配置（推荐）

在 `opencode.json` 中设置：

```json
{
  "agent": {
    "build": {
      "prompt": "你必须使用中文语言思维完成所有任务。在收到用户第一条消息后、执行任何操作或开始推理前，你必须先用中文写一段简短的总结，说明用户需要你做什么、你计划用什么步骤完成、你还缺什么信息。这段中文是你第一个输出，完成这段输出后才能执行具体任务。"
    }
  }
}
```

### 方式二：通过 OhMyOpenAgent prompt_append

在 `oh-my-openagent.json` 中设置：

```json
{
  "agents": {
    "sisyphus": {
      "prompt_append": "你必须使用中文语言思维完成所有任务。..."
    }
  }
}
```

### 方式三：不依赖配置文件

如果无法修改配置文件，可以直接在首个对话消息中包含锚定指令：

> "请使用中文语言思维完成所有任务。开始前先用中文总结我的需求。"

## 注意事项

- 工具调用（bash、HTTP 请求等）的返回值通常是英文，会稀释中文锚定
- 建议在工具调用后主动补充中文语境
- Magic Context 汉化可帮助维持上下文的中文 token 密度
